◆ cholesky()

template<typename ResultType = void, Numeric T>
auto maf::math::cholesky ( const Matrix< T > & matrix)
nodiscard

Computes the Cholesky decomposition of a symmetric positive definite matrix.

This function computes the Cholesky factorization (A = LL^T) for a given square, symmetric, positive definite matrix A, where:

  • A is the input matrix.
  • L is a lower triangular matrix with positive diagonal entries.
  • L^T is the conjugate transpose of L (which is just the transpose for real matrices).

A matrix has a Cholesky decomposition if and only if it is symmetric and positive definite. This function checks for symmetry first. It then detects non-positive-definiteness during the computation.

The decomposition is computed using a blocked Cholesky-Crout algorithm for improved cache performance. The implementation is parallelized using OpenMP (for both multi-threading and SIMD vectorization) for further performance gains.

More information: https://en.wikipedia.org/wiki/Cholesky_decomposition

TThe floating point type of the matrix elements (e.g., float, double).

Returns
(Matrix<T>) The lower triangular matrix (L).
Exceptions
std::invalid_argumentif the input matrix is not symmetric, or if it is not positive definite (detected during factorization).

◆ plu()

template<typename ResultType = void, Numeric T>
auto maf::math::plu ( const Matrix< T > & matrix)
nodiscard

Performs a blocked PLU decomposition on a square matrix.

This function computes the PLU factorization (PA = LU) for a given square matrix A, where:

  • P is a permutation matrix (represented by a vector)
  • L is a unit lower triangular matrix
  • U is an upper triangular matrix

The decomposition is computed using a right-looking blocked algorithm for improved cache performance. It employs partial pivoting (row swapping) to ensure numerical stability. The implementation is parallelized using OpenMP for further performance gains on multi-core systems.

More information: https://en.wikipedia.org/wiki/LU_decomposition#LU_factorization_with_partial_pivoting

Template Parameters
TThe floating point type of the matrix elements (e.g., float, double).
Parameters
matrixThe const reference to the square input matrix (A) to decompose.
Returns
A std::tuple containing:
  1. (std::vector<uint32>) The final permutaion of rows.
  2. (Matrix<T>) The unit lower triangular matrix (L).
  3. (Matrix<T>) The upper triangular matrix (U).
Exceptions
std::invalid_argumentif the input matrix is not square.

◆ QR_decompostion()

template<Numeric T>
QRResultType< T > maf::math::QR_decompostion ( const Matrix< T > & A,
bool full_Q = false,
bool full_R = false )
nodiscard

Computes the QR decomposition of a matrix A using Householder reflections.

Parameters
AThe input matrix to decompose.
full_QIf true, returns the full m x m orthogonal matrix Q. If false, returns the thin m x k matrix Q (where k = min(m, n)).
full_RIf true, returns the full m x n upper triangular matrix R. If false, returns the thin k x n matrix R (where k = min(m, n)).
Returns
A QRResultType<T> containing the matrices Q and R of the decomposition.

This function performs QR decomposition using Householder reflections. It iteratively computes Householder reflectors to zero out sub-diagonal elements of A, storing the reflectors in-place. After processing all columns, it extracts the upper triangular matrix R and constructs the orthogonal matrix Q from the stored reflectors. The user can choose to return either the full or thin versions of Q and R based on their needs.

Exceptions
std::invalid_argumentif A is empty or if dimensions are incompatible for decomposition.
